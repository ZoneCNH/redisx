# redisx 身份

## 我是谁

`redisx` 是 FoundationX 的 **Redis 存储扩展模块**，提供 Redis 客户端封装和标准操作接口。

## 我做什么

| 职责 | 说明 |
|------|------|
| Client 接口 | 定义 Redis 操作的标准接口（Get/Set/Del/Exists/...） |
| 连接管理 | 连接池、健康检查、重连策略 |
| 配置模型 | Redis 连接配置的标准化 |

## 我不做什么

| 不是 | 原因 |
|------|------|
| **不是 Redis 实现** | 底层使用 go-redis |
| **不是业务缓存逻辑** | 业务缓存策略由调用方定义 |
| **不是模板源** | 模板生成属于 xlib-standard |
| **不依赖其他存储模块** | 存储扩展之间平级，无编译期依赖 |

## 宪法合规

| 条款 | 遵循方式 |
|------|----------|
| §3.3 | 存储扩展，可依赖 kernel + observex (interface-only) |
| §3.4 | 不依赖 configx、业务域、其他存储扩展 |
| §1 P13 | 存储扩展之间平级协作 |
