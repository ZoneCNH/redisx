# PATCH-RULE-20260605-001 — Redis L2 provider isolation rule

## Rule

Redis L2 packages must not default-connect to a real Redis endpoint. Any real Redis integration must be opt-in, explicitly configured, and separated from fake/provider contract tests.

## Rationale

Default dials create flaky local tests, secret leakage risk, and downstream coupling. The standard factory should make fake-provider tests the normative path and treat real Redis integration as a separate evidence profile.

## Enforcement

- Public testkit must expose a fake Redis provider.
- Default client construction must use an in-memory/fake provider or an explicitly injected provider.
- Tests must fail if they require `REDIS_URL`, `REDIS_HOST`, `localhost:6379`, go-redis imports, or raw `net.Dial` outside an opt-in integration profile.
