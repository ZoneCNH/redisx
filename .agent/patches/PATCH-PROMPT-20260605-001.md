# PATCH-PROMPT-20260605-001 — Redis L2 closeout prompt hardening

## Problem

Previous closeout prompts allowed a generic base-library scaffold to pass partial review without forcing Redis-named contracts, fake-provider evidence, or a REQ-001..014 audit.

## Patch

For Redis L2 standard-factory closeouts, require each worker report to include:

1. Public config binding evidence for `redisx.Options` or the approved equivalent.
2. Redis error identifier coverage for nil, timeout/canceled, network/auth, read-only/loading/try-again, cluster moved/ask, closed, invalid config, and provider errors.
3. Fake-provider test evidence proving no default Redis connection, even when Redis-like environment variables are present.
4. REQ-001..014 status table with exact file references and verification commands.

## Acceptance

A closeout prompt is acceptable only when it asks for contract files and tests by path, and explicitly forbids default real Redis dials.
